# Demo Flow

1. Upload runbook.
2. Send alert.
3. Send duplicate alert.
4. Analyze incident.
5. Approve remediation.
6. Resolve incident.
7. View audit logs.

## Expected Narrative

An operator uploads a runbook that describes investigation and remediation steps. A new alert arrives and creates an incident. A duplicate alert arrives and is attached to the existing incident rather than creating noise. The operator requests AI analysis, reviews retrieved runbook context and suggested actions, approves remediation, resolves the incident, and verifies the workflow through audit logs.
